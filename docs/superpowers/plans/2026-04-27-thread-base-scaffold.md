# thread-base Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold an empty, working Vue 3 + TS + Bun + Supabase project named `thread-base` with auth infrastructure (no views) and tooling fully wired up.

**Architecture:** Vite-based SPA with Vue 3 Composition API. Three-layer auth: Supabase client singleton (`lib/`), Pinia store (`stores/`), `useAuth()` composable (`composables/`). Router guard `requireAuth` waits on auth init. `main.ts` awaits `auth.initialize()` before mount. Local Supabase via CLI (Docker), generated DB types committed.

**Tech Stack:** Vue 3, TypeScript, Vite, Bun, Tailwind CSS v4, Pinia, Vue Router 4, oxlint, Prettier, Vitest, `@supabase/supabase-js`, Supabase CLI.

**Spec:** `docs/superpowers/specs/2026-04-27-thread-base-scaffold-design.md`

- [ ] **Step 3: Implement `src/stores/auth.ts`**

```ts
import { defineStore } from 'pinia'
import type { Session, User } from '@supabase/supabase-js'
import { supabase } from '@/lib/supabase'

interface AuthState {
  user: User | null
  session: Session | null
  loading: boolean
  initialized: boolean
}

export const useAuthStore = defineStore('auth', {
  state: (): AuthState => ({
    user: null,
    session: null,
    loading: true,
    initialized: false,
  }),

  getters: {
    isAuthenticated: (state) => state.user !== null && state.session !== null,
  },

  actions: {
    async initialize() {
      if (this.initialized) return
      this.initialized = true

      const { data } = await supabase.auth.getSession()
      this.session = data.session
      this.user = data.session?.user ?? null

      supabase.auth.onAuthStateChange((_event, session) => {
        this.session = session
        this.user = session?.user ?? null
      })

      this.loading = false
    },

    async signOut() {
      await supabase.auth.signOut()
    },
  },
})
```

```ts
import type { NavigationGuard } from 'vue-router'
import { useAuthStore } from '@/stores/auth'

export const requireAuth: NavigationGuard = async (to) => {
  const auth = useAuthStore()
  await auth.initialize()
  if (!auth.isAuthenticated) {
    return { name: 'login', query: { redirect: to.fullPath } }
  }
}
```

`auth.initialize()` is idempotent (Task 9), so calling it from the guard is safe even after `main.ts` has already awaited it.

````markdown
Vue 3 + TypeScript + Bun + Supabase scaffold.

1. Install dependencies:
   ```sh
   bun install
   ```
````

2. Start the local Supabase stack (downloads Docker images on first run):
   ```sh
   bun run db:start
   ```
3. Copy environment template and fill in values:
   ```sh
   cp .env.example .env.local
   bun run db:status   # prints API URL + anon key
   ```

4. Generate database types:
   ```sh
   bun run db:types
   ```
5. Start the dev server:
   ```sh
   bun run dev
   ```
